FlowMaps: Modeling Long-Term Multimodal Object Dynamics with Flow Matching

1Sapienza University of Rome 2Université de Montréal 3Mila – Quebec AI Institute

Overview of FlowMaps. A latent flow matching model that, given the current scene Mτ, a queried object label lq and a future time τf, transports a Gaussian latent into a distribution over the object's plausible future bounding boxes. The decoded predictions guide a robot toward where the object is likely to be found.

Abstract

Joint spatial and temporal understanding of 3D scenes is a crucial requirement for robots deployed in everyday household environments. Such agents must not only comprehend and navigate spatial layouts, but also reason about how these spaces evolve over time. In particular, humans interact with objects daily, causing them to change position throughout the environment and making it difficult for robots to reliably associate current observations with previously seen objects. However, these interactions are not random: human habits and routines induce spatio-temporally consistent patterns in object locations, which robotic agents can potentially learn and then exploit for downstream tasks such as navigation. To this end, we introduce FlowMaps, a latent flow matching model for estimating multimodal distributions over the future locations of dynamic objects in a continuous 3D space. By learning the implicit dependencies among objects and their temporal evolution, FlowMaps predicts likely changes in object locations conditioned on past human interactions, while supporting generalization across previously unseen environments that share similar object routines. To demonstrate the utility of this method, we deploy FlowMaps in a downstream dynamic Object Navigation task in both simulated and real-world environments. Across more than 600 episodes, FlowMaps outperforms state-of-the-art approaches, showing that modeling object dynamics through continuous, multimodal spatio-temporal distributions improves robotic search and navigation in changing household environments.

Method

FlowMaps approximates the intractable posterior over an object's future location with flow matching in a learned latent space. It is composed of two modules:

  • A Variational Autoencoder (VAE) that encodes each object token (a normalized 3D bounding box and a semantic label) into a compact latent code, providing the target space for the flow.
  • A latent Conditional Diffusion Transformer (CDiT) that learns a velocity field transporting a Gaussian latent to the latent of the queried object's future bounding box. A map encoder aggregates the scene into context tokens, and a stack of CDiT blocks iteratively refines the noisy query latent by cross-attending to that context.

    • Left: the permutation-invariant map encoder turns the scene into context tokens Hτ. Right: a CDiT block refines the query latent via adaLN-modulated cross-attention to the scene context.

    Learning from Human Habits

    Object movements are not random — they follow recurring human routines that produce semantically consistent patterns. We use ProcTHOR to generate dynamic indoor environments driven by three representative habits: location preferences (returning to a few favoured places), a balanced routine (time spread uniformly across locations), and a highly dynamic routine (frequent transitions). A separate FlowMaps model is trained per habit.

    The location of a single object (a book) across weeks under each of the three modeled habits.

    Downstream Task: Object Navigation

    We deploy FlowMaps on dynamic Object Navigation (ObjNav), our primary downstream application. Given a queried object, FlowMaps predicts where it is likely to have moved and directs the robot toward the most probable locations, recovering gracefully from failed retrievals by reasoning over the full multimodal distribution.
